# Web Design Enhancer V3 (3.0.0a1)

**Principle:** the model proposes and implements; the orchestrator authorizes, verifies, and blocks.

**Stabilization status (Phase 11):** core DoD met on `main` — 512 tests, clean `pip install -e .`, schema `3.0`, three adapters, 12-task corpus (smoke procedural), thin skill + V2 arsenal. Multi-model medians remain lab work (V3.1+), not a ship gate for this alpha.

Full plan: [`docs/superpowers/specs/2026-07-10-wde-v3-plan.md`](superpowers/specs/2026-07-10-wde-v3-plan.md)

## Install (dev)

```bash
cd web-design-enhancer-pro
pip install -e .
wde --help
# without console script on PATH:
python -m wde --help
# from repo root without install (PYTHONPATH=.):
python wde.py --help
```

## First commands

```bash
wde init                 # .wde/ + brief + experience stubs
wde status --json
wde next
wde doctor
wde validate intent
wde validate experience
wde validate design
wde validate lock
wde run static
wde run full --url http://localhost:5173
wde deliver-check
wde review --emit-package --url http://localhost:5173
wde review --url http://localhost:5173
wde report               # consolidated JSON + MD under .wde/reports/
wde benchmark            # smoke corpus; never auto-delivers
wde migrate-v2           # import V2 projects safely
```

## Control plane (per user project)

```text
.wde/
  project.json
  state.json
  capabilities.json
  evidence/
  reports/
  logs/
  cache/
CREATIVE-BRIEF.md
```

`state.json` is owned by **wde-core**. Agents must not hand-edit `phase: READY_TO_DELIVER`.

## Relationship to V2 skill

V2 `SKILL.md` + `scripts/*` remain the validation arsenal. V3 wraps them in a **stateful orchestrator** so models cannot claim gates without fresh proof.

Tagged freeze of V2 line: **`v2.2.0`**.
