# Web Design Enhancer V3 (alpha)

**Principle:** the model proposes and implements; the orchestrator authorizes, verifies, and blocks.

Full plan: [`docs/superpowers/specs/2026-07-10-wde-v3-plan.md`](superpowers/specs/2026-07-10-wde-v3-plan.md)

## Install (dev)

```bash
cd web-design-enhancer-pro
pip install -e .
# or without install:
python -m wde.cli.main --help
```

## First commands

```bash
python -m wde.cli.main init                 # .wde/ + brief + experience stubs
python -m wde.cli.main status --json
python -m wde.cli.main next
python -m wde.cli.main doctor
python -m wde.cli.main validate intent
python -m wde.cli.main validate experience
python -m wde.cli.main validate design
python -m wde.cli.main validate lock
python -m wde.cli.main run static
python -m wde.cli.main run full --url http://localhost:5173
python -m wde.cli.main deliver-check
python -m wde.cli.main review --emit-package --url http://localhost:5173
python -m wde.cli.main review --url http://localhost:5173
python -m wde.cli.main report               # consolidated JSON + MD under .wde/reports/
python -m wde.cli.main benchmark            # smoke corpus; never auto-delivers
python -m wde.cli.main migrate-v2           # import V2 projects safely
```

## Control plane (per user project)

```text
.wde/
  project.json
  state.json
  capabilities.json
  evidence/
  reports/
  logs/
  cache/
CREATIVE-BRIEF.md
```

`state.json` is owned by **wde-core**. Agents must not hand-edit `phase: READY_TO_DELIVER`.

### Run mechanical suite

```bash
wde run static --root .
wde run mechanical --root .
wde run browser --root . --url http://localhost:5173
wde deliver-check --root .
```

## Relationship to V2 skill

V2 `SKILL.md` + `scripts/*` remain the validation arsenal. V3 wraps them in a **stateful orchestrator** so models cannot claim gates without fresh proof.

Tagged freeze of V2 line: **`v2.2.0`**.
